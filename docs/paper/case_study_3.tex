\subsection{Case Study 3: EEE Separates Reproducibility Failures from Non-Comparable Runs}%
\label{sec:cs3-helm}

We use HELM to test whether \eee{} supports instance-level reproducibility
  analysis beyond aggregate score comparison.
We re-executed Pythia-6.9B, Vicuna-7B v1.3, and Falcon-7B on fourteen
  single-turn HELM benchmarks and compared public records to local
  reproductions~\citep{helm,pmlr-v202-biderman23a,vicuna2023,almazrouei2023falcon}.
We converted both sides to \eee{} and joined records by \texttt{sample\_hash}, a
  hash of the rendered prompt and reference.
Each cell reports the micro-average agreement: the fraction of matched
  \mbox{(instance, core metric)} score pairs for which the public and local
  scores are equal up to \texttt{abs\_tol=1e-9}.
Figure~\ref{fig:cs3-helm-heatmap} reveals four non-reproducibility patterns:
  non-comparability, serving artifacts, stochastic disagreement, and residual
  answer differences.

\paragraph{Entity-Matching is non-comparable, not disagreeing.}
Each side contains 1{,}000 Abt--Buy~\cite{kopcke2010evaluation} examples, but
  the \texttt{sample\_hash} sets differ, meaning each run was scored on a
  different subset despite using the same recipe.
A pandas merge row-order change in HELM scenario
  construction~\citep{mckinney2010data,pandas_merge_docs} permuted the candidate
  set before fixed-seed downsampling, selecting different prompts.
Because the rendered prompts and references do not match, \eee{} rejects the
  comparison and reports the cell as \texttt{join\_failed}.

\paragraph{SyntheticReasoning-Natural exposes a Pythia serving artifact.}
This discrepancy emerges because the \texttt{output.raw} field in the public
  HELM results is always empty, and thus the score is always zero.
In contrast, our reproduction of this benchmark shows completions with low but
  non-zero scores.
Lower-level HELM artifacts show that the public run produced outputs, but each
  began with the newline stop sequence and was trimmed before scoring.
This likely reflects a deployment difference: HELM v0.3.0 routed the public
  Pythia run through Together.ai, whereas ours was run locally via the Hugging
  Face deployment.
\eee{} does not encode the historical serving stack, but it surfaces the
  symptom: empty completions in the scored records.

\paragraph{WikiFact disagreement is recipe-level stochasticity.}
The WikiFact cells cluster near 0.92 because the recipe uses
  \texttt{temperature=1.0} and \texttt{num\_outputs=5}.
This is slightly above the homogeneous-Bernoulli prediction at the mean accuracy
  (Appendix~\ref{sec:app-helm-wikifact}), consistent with prompt-dependent
  difficulty.

\paragraph{Residual disagreements are real but smaller.}
For the remaining cells below one, the runs usually evaluate the same instances
  and mostly agree but differ on a small number of generated answers,
  concentrated in Pythia-6.9B.
Pythia may be more sensitive to serving-stack changes, but the audit cannot
  separate model weakness from deployment details.
Thus, \eee{} detects several failure modes and provides forensic leads, but does
  not replace framework-level provenance.
